# /onboard — personalized setup from CV + job posting

## Goal
Extract skill requirements from a job posting, compare against user's CV, generate a personalized learning plan with topic selection.

## Procedure

### Step 1 — Collect inputs

Major (in persona, but brief):
> "NOWY REKRUT. Zanim zaczniemy — dwa dokumenty na stół."

Ask for (one at a time):
1. "Wklej swoje CV (tekst lub ścieżka do pliku PDF/MD)."
2. "Wklej treść oferty pracy na którą aplikujesz."

If user provides a file path — read it. If PDF — read with Read tool (supports PDF).

### Step 2 — Analyze job posting

Extract from job posting:
- **Required technologies** (languages, frameworks, tools, databases, protocols)
- **Required domain knowledge** (e.g., e-commerce, fintech, healthcare)
- **Required soft skills** (for mock interview focus)
- **Seniority level** (junior/mid/senior/lead — affects Bloom level targeting)
- **Company name** and **role title**

### Step 3 — Analyze CV

Extract from CV:
- **Known technologies** with approximate proficiency (years of use, project context)
- **Experience years**
- **Domain experience**
- **Primary stack**

### Step 4 — Gap analysis

For each required technology/skill from job posting:
- If present in CV with significant experience → priority: low (refresh)
- If present in CV but shallow → priority: normal (strengthen)
- If absent from CV but required → priority: high
- If absent from CV and prominently featured in posting (e.g., "must have", appears 3+ times) → priority: critical

Generate topic list as JSON internally for processing.

### Step 5 — Present multi-pick to user

Show the generated list with priorities and reasoning:

```
ANALIZA GOTOWA. Oto co wyciągnąłem:

Firma: $COMPANY — $ROLE
Wymagania vs twoje CV:

 # | Temat              | Priorytet | Powód
---|--------------------|-----------|-----------------------------------------
 1 | Kotlin             | CRITICAL  | Wymagane w ofercie, brak na CV
 2 | Coroutines         | CRITICAL  | "must have" w ofercie
 3 | Spring Boot        | NORMAL    | Masz Spring, ale posting wymaga Boot 3
 ...

Które tematy bierzemy? (numery oddzielone przecinkami, lub "all" dla wszystkich)
```

Wait for user response. Parse selection.

### Step 6 — Generate topics

For each selected topic:
1. Generate a slug: lowercase, snake_case, ASCII-only (e.g., "Spring Security" → `spring_security`)
2. Check if `content/topics/<slug>.md` exists
3. If YES — register in topics.json with extracted priority
4. If NO — generate full topic file using the same procedure as `/more` skill Step 3:
   - 5 questions per Bloom level (20 minimum, 32 for critical/high)
   - Full model answers, interview traps, tags
   - Scope section with 8-15 subtopics
   - Context line referencing the target company/role
   - Header: `> Auto-generated by Major. Review welcome.`

Also check if `content/coding/<slug>.md` exists — if not, generate a starter coding task bank (4-7 tasks, beginner track).

Register all in `state/topics.json` with structural mastery:
```json
"<slug>": {
  "priority": "<from gap analysis>",
  "mastery": {
    "theory": 0.0,
    "coding": {"junior": 0.0, "mid": 0.0, "senior": 0.0}
  },
  "status": "queued",
  "due": null,
  "questions_asked": 0,
  "last_practice": null
}
```

### Step 7 — Fill learner profile

Write `state/learner_profile.json`:
```json
{
  "name": "private",
  "goal": "Job interview — $ROLE at $COMPANY",
  "interview_date": null,
  "interview_company": "$COMPANY",
  "interview_company_notes": "<extracted from posting>",
  "experience_years": "<from CV>",
  "primary_stack": "<from CV>",
  "self_assessment": {"<slug>": "<estimated 0-5 from CV analysis>"},
  "sessions_planned": null,
  "session_minutes": 60,
  "preferences": {"pace": "fast", "style": "drill_then_lesson"},
  "language": "polish",
  "created_at": "<ISO>",
  "updated_at": "<ISO>"
}
```

Ask user:
- "Kiedy rozmowa? (data lub 'nie wiem')"
- "Ile minut na sesję? (domyślnie 60)"
- "Język: polski czy angielski?"

Update profile with answers.

### Step 8 — Build study track

Order topics by: critical first, then high, normal, low. Within same priority: lowest self_assessment first.

Write `study_track` array to `topics.json`.

Set `job_context` in `topics.json` from posting analysis (one-line summary of company + role + stack).

### Step 9 — Handoff

> "PLAN GOTOWY. $N tematów na celowniku, $CRITICAL_COUNT krytycznych. Wpisujesz /start i zaczynamy od $FIRST_TOPIC. RUSZAJ."

Set up `state/current.json` for first session.

## Important

- Do NOT ask all questions at once — go step by step.
- CV analysis should be generous (benefit of the doubt on claimed skills).
- Job posting analysis should be thorough (catch implicit requirements like "experience with microservices" implying Docker/K8s knowledge).
- If user doesn't have a CV or job posting, suggest `/start` for manual onboarding or `/more` to add topics individually.
